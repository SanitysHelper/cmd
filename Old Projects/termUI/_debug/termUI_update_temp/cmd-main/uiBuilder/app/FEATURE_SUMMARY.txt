PROGRAM: PowerShell UI Menu Builder
Feature Implementation Summary
========================================================================

CORE FEATURES
  * Numbered menu interface with selection (1, 2, 3...)
  * Interactive menu with multi-level submenu navigation
  * Hierarchical menu structure stored in CSV format
  * Path-based naming convention (mainUI.category.action)
  * Exit code routing (1-N for selections, 99 for quit)
  * File output (selection path written to run_space/selection.txt)
  * Console confirmation output ("Selected: path")

USER EXPERIENCE
  * Fully piped input support (zero manual typing required)
  * Numbered selection with validation (1-N + 0=back + q=quit)
  * Automatic submenu navigation on selection
  * Graceful back button (navigation stack)
  * Graceful quit (exit code 99)
  * Color-coded menu display
  * Clear error messages with prefixes

ARCHITECTURE
  * Main executable: run.bat (batch launcher)
  * Implementation: UI-Builder.ps1 (~770 lines, monolithic)
  * Data storage: button.list (CSV: Name,Description,Path,Type)
  * Configuration: settings.ini with [General], [Colors], [Logging]
  * Temp workspace: run_space/ for execution artifacts
  * Pure functions with no side effects (except logging/I/O)
  * Single hashtable ($buttonIndex) for O(1) path lookup
  * Breadcrumb array for navigation history

CLI COMMANDS
  * add <name> <desc> <path> <type>: Add menu item
  * remove <path>: Remove menu item
  * list: Display all buttons with type indicators
  * --generate-stub <language>: Generate code stubs (ps1|bat|py|cs)
  * --debug: Enable debug mode with verbose logging

CORE FUNCTIONS (Implemented & Tested)
  * Read-ButtonList: Parse CSV to menu structure
  * Write-ButtonList: Serialize menu back to CSV
  * Get-ChildButtons: Filter and sort children by parent
  * Add-ButtonOption: Insert new item and persist
  * Remove-ButtonOption: Delete item and persist
  * Show-NumberedMenu: Display options and capture input
  * Invoke-OutputSelection: Write file, console, exit code
  * Invoke-MainLoop: Main interactive loop with navigation
  * Get-LanguageStub: Generate code for 4 languages

LOGGING & MONITORING
  * Log location: _debug/logs/
  * Log types: navigation.log, input.log, error.log, important.log
  * Debug logging: ui-debug.log (debug mode only)
  * Configurable per log type via settings.ini
  * ISO 8601 timestamps on all entries
  * Non-blocking (doesn't interrupt menu flow)

LANGUAGE INTEGRATION
  * PowerShell stub: Read file, switch on selection
  * Batch stub: setlocal, for loop, conditional
  * Python stub: open(), read(), if/elif structure
  * C# stub: File.ReadAllText(), switch statement
  * Each stub includes example function calls
  * Pattern: Launch menu -> Read selection.txt -> Call handler

PIPED INPUT HANDLING (Critical Architecture Decision)
  * Detects if stdin is redirected: [Console]::IsInputRedirected
  * Interactive: Uses Read-Host for console prompts
  * Automated: Uses [Console]::In.ReadLine() for pipe
  * Gracefully handles EOF (exhausted pipe -> quit with code 99)
  * Multi-step sequences fully supported
  * Example: @("2","1") -join "`n" | .\run.bat
